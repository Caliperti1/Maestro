# Contributing

Maestro is currently a personal MVP project. Contributions should map to an issue or explicitly approved work package.

## Workflow

1. Pick or create a GitHub issue.
2. Create a focused branch.
3. Keep changes scoped to the issue.
4. Update docs when architecture or behavior changes.
5. Run relevant tests or explain why they are not available yet.
6. Open a draft PR for user testing.

## Commit Hygiene

- Do not commit secrets.
- Do not commit generated cache files.
- Keep commits understandable and reversible.
